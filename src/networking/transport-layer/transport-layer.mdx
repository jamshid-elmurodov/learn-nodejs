# Transport layer
- TCP: datani yuborilganligini tekshiradi, agar fail bo'lgan bo'lsa yana qayta yuboradi shuning uchun ham xavfsiz.
       va data ning tartibini nazorat qiladi, shuning uchun ham u sekinroq, foydalanish maqsadlari: http, https, ftp, smtp 
    - Source Port | Qaysi portdan yuborilgan
    - Destination Port | Qaysi portga borishi kerak
    - Sequence Number | Data tartibini saqlash uchun
    - Acknowledgment Number | Oldingi paket olinganini tasdiqlash
    - Header Length | TCP headerning uzunligi
    - Flags | SYN, ACK, FIN, RST kabi boshqaruv bitlari
    - Window Size | Qancha data qabul qilishga tayyorligi
    - Checksum | Xatoliklarni aniqlash
    - Urgent Pointer | Zudlik bilan uzatiladigan data borligini bildiradi
    - Options | Qo‘shimcha konfiguratsiya (masalan, MSS, timestamps)

- UDP: data yuboradi va uning muvaffaqiyatli yuborilganligini tekshirmaydi, shunig ba'zida data yo'qolishi mumkin.
       shuning uchun ham u tez, foydalanish maqsadlari: video call, gaming, live stream 
    - Source Port | Qaysi portdan yuborilgan
    - Destination Port | Qaysi portga borishi kerak
    - Length | Butun UDP paketining uzunligi (header + data)
    - Checksum | Xatolikni aniqlash (lekin ba’zi OS'larda ixtiyoriy)
